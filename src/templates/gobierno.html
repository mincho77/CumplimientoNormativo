Estructura de Gobierno y Consulta para el Cumplimiento

Este marco define el gobierno y ordenamiento de los datos de cumplimiento regulatorios para SURA.

1. Taxonomía de Clasificación de Normas

Cada obligación detectada por el sistema se asocia bajo la siguiente jerarquía para permitir consultas dirigidas:

1. Regulador / Emisor (e.g. SFC)
2. Compañía Impactada (e.g. ARL Sura)
3. Macroproceso (e.g. Gestión de la solidez)
4. Proceso (e.g. Gestión integral de reservas)
5. Líder Responsable (e.g. Maria del Pilar Trujillo)

2. Estructura de Almacenamiento

  • Inventario Consolidado (Base de Datos): Se almacena en la tabla consolidada data/tablero_semaforo.csv. Contiene cada obligación con su respectivo estado de semáforo de riesgo y campos de control SOX.
  • Trazabilidad en Obsidian (Wiki): Cada estándar analizado genera un documento Markdown en wiki/sources/ con links a las entidades y conceptos correspondientes.

3. Flujo de Trabajo Recomendado

  1. Monitorear: Revisar nuevas alertas diarias en el dashboard principal.
  2. Analizar: Ejecutar el análisis con IA sobre los PDFs. El sistema asocia automáticamente la obligación con uno de los 94 macroprocesos y su líder responsable.
  3. Registrar: Presionar "Guardar en Wiki" para archivar y actualizar la base de datos de control.
